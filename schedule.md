# CKB Builder Track — 12-Week Curriculum

A progressive study plan for the CKB Builders' track. Workload starts light on theory and ramps up into advanced hands-on work by the final month.

---

## Phase 1 — Introduction (Weeks 1–2)

### Week 1 · Theory & orientation `● ○ ○ ○ ○`

**This week**

- Read the intro to Nervos CKB docs (technical concepts + terminology)
- Watch Nervos Nation explainer videos (UTXO/PoW, Cell Model, RISC-V)
- Read: Nervos layered blockchain overview + CKB ethos articles
- Set up your GitHub dev log repo — start documenting from day one

**Log by end of week:** Dev log repo created, 2–3 reading notes committed

---

### Week 2 · Environment setup + CKB basics `● ● ○ ○ ○`

**This week**

- Install OffCKB + set up your dev environment
- Work through the Quick Start guide
- Complete CKB Academy lessons 1 and 2
- Read: Introduction to Script (smart contracts)
- Skim the tokenomics deep-dive and Bitcoin vs CKB security model articles

**Log by end of week:** Dev environment running, Academy lessons done, notes logged

---

## Phase 2 — Beginner (Weeks 3–5)

### Week 3 · First practical tutorials `● ● ○ ○ ○`

**This week**

- Tutorial: Transfer CKB — retain screenshots for your dev log
- Tutorial: Store Data on Cell
- Get testnet CKB from the faucet
- Read: Account abstraction + Web5 articles from the reading list

**Log by end of week:** 2 tutorials completed with screenshots in dev log

---

### Week 4 · Tokens + digital objects `● ● ● ○ ○`

**This week**

- Tutorial: Create Fungible Token
- Tutorial: Create a DOB (digital object / Spore)
- Explore the CCC App docs and test snippets in the CCC Playground
- Skim CKB Tools website to get familiar with the ecosystem

**Log by end of week:** Token and DOB tutorials done, first CCC playground experiments

---

### Week 5 · Locks + app ideation `● ● ● ○ ○`

**This week**

- Tutorial: Build a Simple Lock
- Work through the CCC code examples and API reference
- Start ideating your beginner app — discuss with programme director
- Read: L1 developer course theory sections (skip Lumos/Capsule exercises)

**Log by end of week:** All 5 beginner tutorials complete; app idea drafted

---

## Phase 3 — Intermediate (Weeks 6–9)

### Week 6 · Script course: validation + basics `● ● ● ○ ○`

**This week**

- Script Course Class 1: Validation Model
- Script Course Class 2: Script Basics
- Script Course Class 3: UDT
- Begin building your basic app (first working skeleton)

**Log by end of week:** Classes 1–3 complete; app skeleton committed to dev log

---

### Week 7 · Script course: WASM + debugging `● ● ● ● ○`

**This week**

- Script Course Class 4: WebAssembly on CKB
- Script Course Class 5: Debugging
- Script Course Class 6: Type ID
- Continue building your app — add core functionality
- Use CKB Debugger on any issues encountered

**Log by end of week:** Classes 4–6 complete; app core functionality working

---

### Week 8 · Script course: advanced + sUDT `● ● ● ● ○`

**This week**

- Script Course Classes 7–10 (Duktape, Performant WASM, Cycle Reductions, Language Choices)
- Read the sUDT standard document
- Work through the sUDT CLI tutorial
- Finish and polish your beginner app — screenshot proof + dev log writeup

**Log by end of week:** Script course complete; beginner app published + documented

---

### Week 9 · Nervos DAO + Spore deep dive `● ● ● ● ○`

**This week**

- Read Nervos DAO introduction and deposit/withdraw RFC
- Study NervDAO browser implementation source code
- Read through comprehensive Spore documentation
- Explore Spore SDK + DOB Cookbook examples
- Detailed JS scripting guide (quick start)

**Log by end of week:** DAO and Spore concepts documented in dev log

---

## Phase 4 — Advanced (Weeks 10–12)

### Week 10 · xUDT + SSRI intro `● ● ● ● ○`

**This week**

- Read xUDT introduction + RFC
- Read "Enhance sUDT programmability with xUDT" blog post
- Read SSRI introduction — understand the motivation and architecture
- Study the Pausable UDT repo as a full SSRI-compliant example
- Detailed Rust scripting guide (if using Rust)

**Log by end of week:** xUDT vs sUDT comparison + SSRI notes in dev log

---

### Week 11 · RGB++ + iCKB `● ● ● ● ●`

**This week**

- Read RGB++ introduction and light paper
- Explore RGB++ Resource Hub (Script Standard, SPV, API tools)
- Read iCKB whitepaper and "Unlocking CKB Liquidity" article
- Study iCKB SDK and audit report
- Begin planning an intermediate/advanced project idea

**Log by end of week:** RGB++ and iCKB notes committed; advanced project idea drafted

---

### Week 12 · Ecosystem scripts + capstone `● ● ● ● ●`

**This week**

- Study ecosystem scripts and libraries overview
- Review Molecule / serialization documentation
- Revisit the Nervos RFCs GitHub for any open proposals relevant to your project
- Explore Spark programme and CKB Community Fund DAO for funding potential
- Final dev log review — ensure all tutorials, notes, and projects are well documented

**Log by end of week:** Dev log polished + advanced project proposal or prototype ready

---

## Notes

- **Dev log**: Start in week 1 and commit every week without exception. Screenshots, notes, and reflections all count. A well-maintained dev log is evidence of participation and can support a Spark or Community Fund application.
- **App builds**: The beginner app (ideated in week 5, shipped in week 8) should be something genuinely useful or interesting — token minter, Spore generator, or something more ambitious if you have prior development experience. Discuss scope with your programme director early.
- **Advanced project**: Begin forming this idea in week 11. By week 12 you should have either a working prototype or a detailed proposal ready to share.
- **Deprecated tools**: Some older resources reference Lumos or Capsule — these are deprecated. Use CCC and OffCKB instead.
